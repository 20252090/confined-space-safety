# 밀폐공간 안전관리 앱

맨홀 등 밀폐공간에서 작업하는 고위험 작업자의 상태를 **관리자**가 현장에서 실시간으로 모니터링하고 입출입·안전점검을 기록하기 위한 웹 앱입니다.

- 순수 **HTML / CSS / JavaScript** (프레임워크·빌드 의존성 없음)
- 데이터는 브라우저 **localStorage** 에 저장
- 모바일(스마트폰/태블릿) 화면 기준, 장갑 착용·야외 조작을 고려한 큰 버튼·고대비 UI

## 화면 구성 / 흐름

앱은 **스플래시 → 로그인 → 홈** 순으로 진입합니다. 스플래시(부팅) 후 **관리자 이름·소속·작업 현장**을 선택해 로그인하면 **홈 화면**이 나오고, 홈에서 현황/준비/기록으로 이동합니다. 하단 탭은 **홈 · 현황 · 준비 · 기록** 4개이며, 설정은 상단 ⚙ 아이콘으로 진입합니다. 상단에는 모든 화면 공통으로 **상시 상태 스트립**(내부 인원·정상/주의/위험 집계·현재 화면)이 노출됩니다. 전체 화면은 **태블릿 프레임** 안에 표시됩니다.

| 탭 / 진입 | 내용 |
|----|------|
| **로그인** | 관리자 이름·소속(선택)·작업 현장을 선택해 로그인. 세션은 localStorage에 저장되어 재접속 시 유지되며, 홈의 `로그아웃 · 현장 변경`으로 해제 |
| **홈** | 로그인 후 진입 화면. 관리자·작업 현장·날짜, 내부 인원과 정상/주의/위험 집계, 현황·준비·기록 바로가기 |
| **현황 (관제 뷰)** | 하수관로 **내부 종단면도**(지표면·지층·심도축·맨홀 수직구/사다리·관로/물·관경·경사·관저고) 위에 작업자를 **심도 위치**에 마커로 표시. 좌상단 현장정보(내부 인원·상태 집계·대표 가스 O₂·H₂S·CO·LEL), 우상단 경고 패널(위험/주의/정상 카운트·경보등급 필터·경보 리스트). **작업자 선택 시 AR 글라스 카메라 뷰**(맨홀 내부 1인칭 POV + HUD: O₂ 게이지·가스 바·심박/체온/경과/심도·상태/경고·무전/구조/복귀). 가스·바이탈은 시뮬레이션 값 |
| **준비** | ① 입장 전 안전점검(가스측정·환기·송기마스크·안전대·무전기·작업허가서 6항목) + ② 작업자 입장을 한 화면에 통합. 작업자 등록 시 **맨홀 번호(MH-01~10)** 를 지정하며(이미 작업 중인 맨홀은 선택 불가), 6항목을 모두 완료해야 입장 버튼이 열림(관문). 입장/퇴장 시각 자동 기록 |
| **기록** | 날짜별 작업 일지(맨홀 포함), 경보 발생 이력 |
| **설정 (⚙)** | 안전확인 주기/유예, 가스·생체 임계치, 가스 시뮬레이션 모드, 데이터 내보내기/초기화 |

## 작업 흐름 (운영 시나리오)

1. **등록**: 작업자 이름 + **맨홀 번호**를 지정해 등록한다.
2. **안전점검**: 6항목을 모두 완료해야 입장이 열린다(미완료 시 입장 차단).
3. **입장 → AR·태블릿 연결**: 입장하면 `AR 연결 중` 상태로 대기하다가 연결 완료 시 자동으로 작업(하강·현황)을 시작한다. 안전확인 주기는 연결(작업 시작) 시점부터 카운트된다.
4. **현황**: 관제 뷰에서 심도·관로 구조 위 작업자 위치와 심박·SpO₂·체온·가스를 실시간 파악한다.
5. **무전**: 작업자 마커를 눌러 AR 뷰의 `무전`으로 지시를 전달/수신한다(프리셋: 위치 보고·가스 재측정·작업 중지·즉시 퇴장). 안전확인 유예를 넘기면 관제가 자동으로 무전을 호출하며, **작업자의 무전 응답**이 오면 안전확인이 갱신된다(관리자가 대신 확인 처리하지 않음). 현재 응답은 시뮬레이션.
6. **복귀**: `복귀` 버튼을 누르면 작업자가 맨홀 위(지표면)로 **천천히 상승**하고, 지표면에 닿으면 완전 복귀로 확정되어 입출입 기록이 마감된다.

## 안전 로직

- **안전 확인(응답 주체 = 작업자)**: 안전확인 응답은 **작업자만** 한다. 관제(관리자)가 대신 눌러 확인 처리하지 않으며, 작업자의 **무전 응답**이 오면 안전확인이 갱신되어 정상으로 복귀한다.
  - **단계적 대응**: 작업 시작(연결) 후 설정 주기(기본 5분)가 지나면 ① **`안전확인 요청`(주의)** → 유예(기본 60초) 초과 시 ② **`무전 확인`** (관제가 자동으로 무전 호출) → 그래도 무응답이면 ③ **`퇴장 권고`** 로 올라간다. 여기까지는 아직 위험(구조)이 아니다.
  - **119 격상 조건**: 퇴장 권고에도 응답이 없고 **작업자의 움직임까지 사라지면(무움직임)**, 비로소 **위험 → `119 연락·구조`** 로 자동 격상된다(무응답 **그리고** 무움직임이 함께여야 함).
- **생체신호**: 심박·체온·SpO₂ 중 하나라도 임계치를 넘으면 **주의 → 퇴장 권고**, 위험 → **구조 알림(자동)**.
- **가스**: 임계치에 **근접하면 주의(퇴장 권고)**, **초과하면 위험(연락 + 구조 알림 자동)** 으로 전환되고 경보 이력에 기록된다.
- **구조 알림 · 119 연계**: 가스·생체 위험은 자동, 안전확인 미응답은 `구조` 버튼으로 수동 발령된다. 구조 알림은 **전체 화면을 가리지 않는 하단 카드**로 떠서 관제 화면을 계속 볼 수 있다. **119 전달 정보**(위치·맨홀·심도·상황·가스·생체·경과·연락처)를 자동 정리해 보여주고, `📞 119 신고 전화`(tel:119, 발신 시 이력 기록) 버튼과 **지시자 대응 절차**(① 상태 확인 → ② 119 신고 → ③ 단독 진입 금지 → ④ 환기 유지 → ⑤ 주변 대피 → ⑥ 구조대 인계), 해당 작업자 **무전 교신** 바로가기를 제공한다.
- **SOS / 전체 대피**: 상단 SOS 버튼 → 확인 후 ① **상단(헤더)만 빨간 테마로 점멸**하고(전체 화면은 가리지 않음), ② **119 신고 전화 발신**이 이력에 기록되며 상단 배너에 `📞 119 신고`(tel:119) 버튼이 뜨고, ③ **내부 작업자 전원이 맨홀 위로 상승해 복귀**(대피)하는 모습이 관제 뷰에 그대로 보인다. 배너의 `해제`로 종료.
- **구조 버튼 노출 규칙**: AR 뷰의 `구조` 버튼은 작업자가 주의·위험일 때만 보이며, **정상 상태로 돌아오면 사라진다.**
- 가스·생체 값은 현재 **시뮬레이션** 이며, 설정의 '경보 테스트' 모드로 위험 상황을 재현할 수 있습니다.

## 실행

정적 파일이므로 `index.html` 을 브라우저로 열면 됩니다.

### GitHub Pages 배포
1. 이 폴더를 저장소에 push
2. Settings → Pages → Branch 를 `main` / root 로 지정
3. `.nojekyll` 이 포함되어 있어 별도 설정 없이 그대로 서비스됩니다.

## 파일 구조
```
index.html   화면 구조(로그인·홈 + 상태 스트립 + 4탭 홈/현황/준비/기록 + 설정 + 모달 + 경보 오버레이)
style.css    산업 안전 다크 테마 / 상태색 / 큰 터치영역
app.js       상태관리(localStorage) · 시뮬레이션 · 경보 로직 · 뷰 전환
```
